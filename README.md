# SkladExcel

Складской учет на VBA (Excel).

## Работа с Git без merge conflicts

Чтобы уменьшить конфликты при переносе/слиянии в GitHub:

- В репозитории храним **только текстовый экспорт VBA** (`modules/*.bas`, `forms/*.frm`, `classes/*.cls`, `sheets/*.bas`).
- Файл книги Excel (`*.xlsm`) считается бинарным артефактом и в репозиторий не коммитится.
- Для VBA-файлов зафиксированы окончания строк CRLF через `.gitattributes` — это снижает шум в diff и количество конфликтов.
- `VBA_Sklad_Dump_UTF8.txt` помечен как производный файл: при merge берется версия текущей ветки.

## Warehouse CRUD flows

Рабочие CRUD-потоки складов реализованы в модуле `modules/a_demo.bas` (историческое имя модуля, используется в full-версии):

- `добавить_склад`
- `rename_sk`
- `delete_sk`
- миграция движений при удалении склада
- синхронизация UI-списков складов

## Переход с demo на full

В кодовой базе отключены демо-ограничения для CRUD складов, а сообщения об ограничениях обновлены на нейтральные (без привязки к «Демо-версии»).
